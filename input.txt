# USER STORIES:
1. As a user I would the system to retrieve data from sensors through GET endpoints so that sensor data can be ingested into the system.
2. As a user I would the system connect to telemetry endpoints via REST APIs to continuously receive sensor data.
3. As a user I would scalar data to be converted into a unified internal schema so that it can be processed in a standardized way.
4. As a user I would complex data structures to be converted into the same unified internal schema so that all data types can be processed consistently.
5. As a user I would normalized data to be sent to a message broker so that downstream components can process it asynchronously.
6. As a user I would the automation engine consume broker messages in real time to evaluate rules without causing congestion.
7. As a user I would the system keep the latest value retrieved from each sensor in cache memory so that rules can access the most recent data efficiently.
8. As a user I would the system evaluate rules dynamically whenever a new event arrives.
9. As a user I would define rules using logical comparison operators (<, >, =).
10. As a user I would automatically send a POST request to actuators when a rule condition is satisfied.
11. As a user I would actuation rules to be stored in a database so that they persist across system restarts.
12. As a user I would the automation engine retrieve all rules from the database when the system starts.
14. As a user I would create new rules through a dashboard interface and store them in the database.
15. As a user I would delete or delete individual rules from the database through the dashboard interface.
16. As a user I would clear all rules from the database through the dashboard interface.
17. As a user I would visualize a real-time dashboard displaying the values of all sensors.
18. As a user I would receive real-time updates from the backend to keep the dashboard data current.
19. As a user I would see line charts for critical metrics and the current state of all actuators on the dashboard.
20. As a user I would start the entire system with a single command.
